# Tasks: Netflix RLS — Tableau → Power BI Migration

**Input**: Design documents from `specs/003-netflix-rls-pbi/`
**Prerequisites**: [plan.md](plan.md) (required), [spec.md](spec.md) (required for user stories)
**Design inputs**: `.specify/memory/NetflixRLS/dax-measures-output.md`, `.specify/memory/NetflixRLS/star-schema-output.md`, `.specify/memory/NetflixRLS/tableau-analysis-output.md`

**Tests**: No automated test suite is requested. "Validation" tasks use the mandatory structural validators (`tmdl-validate`, `validate_pbip.py`) plus a functional RLS spot-check (constitution §10 / plan §4).

**Organization**: Tasks are grouped by user story (P1 → P2 → P3) so each story is independently buildable and testable. The build target is the PBIP project at `Output/NetflixRLS/`.

## Format: `[ID] [P?] [Story] Description`

- **[P]**: Can run in parallel (different files, no dependencies on incomplete tasks)
- **[Story]**: US1 = Dynamic RLS, US2 = Worksheet visuals, US3 = Year parameter
- Exact file paths are included in every task.

## Path Conventions

All artifacts are emitted under the repository's `Output/NetflixRLS/` build target (plan §Project Structure). The semantic model uses the TMDL `definition/` folder form with a `roles/` subfolder; the report uses the PBIR enhanced `definition/pages/` folder form.

## Phase 2: Foundational (Blocking Prerequisites)

**Purpose**: The shared semantic-model core (catalog table, date dimension, measures, M load) that **every** user story depends on. RLS-aware counts (US1), all nine visuals (US2), and the time axis all rely on these tables and the `[Total Titles]` measure.

**⚠️ CRITICAL**: No user story work can begin until this phase is complete.

- [ ] T005 Create `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/database.tmdl` with `compatibilityLevel` (1567+) and the model `database` id.
- [ ] T006 Create the model header `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/model.tmdl` (culture `en-US`, default annotations, and `ref table` entries — `netflix_titles`, `DimDate` now; `User_Access` and `Year` added by their stories).
- [ ] T007 Create `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/tables/netflix_titles.tmdl` — the wide secured catalog. Power Query M partition loads `Data/Netflix RLS/netflix_titles.csv` via `Csv.Document(File.Contents(...), [Delimiter=",", Encoding=65001, QuoteStyle=QuoteStyle.Csv])`, `Table.PromoteHeaders(..., [PromoteAllScalars=true])`, cast `show_id`/`release_year` → `Int64.Type` and remaining source columns → `type text`, apply `Text.Trim` to `country` (RLS key), and add the **`Year` Date column** by transforming `date_added` (`"MMMM d, yyyy"`) via `Date.From(DateTime.FromText(Text.Trim(_), [Format="MMMM d, yyyy", Culture="en-US"]))` → `type date`. Declare all 12 source columns plus the `Year` Date column (`formatString` "General Date").
- [ ] T008 [P] Create `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/tables/DimDate.tmdl` — calendar generated by `ADDCOLUMNS(CALENDAR(DATE(2008,1,1),DATE(2025,12,31)), "Year",YEAR([Date]), "Quarter","Q"&FORMAT([Date],"Q"), "Month",FORMAT([Date],"MMMM"), "MonthNumber",MONTH([Date]), "Day",DAY([Date]))`, with a `Calendar` hierarchy (`Year > Quarter > Month > Date`), and marked as a date table on `Date`.
- [ ] T009 Add the 6 explicit measures to `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/tables/netflix_titles.tmdl` (DAX verbatim from `dax-measures-output.md`), each with `formatString` + `displayFolder`: `Total Titles` = `DISTINCTCOUNT(netflix_titles[show_id])` (`#,##0`, Catalog Metrics); `Total Movies` = `VAR _movie="Movie" RETURN CALCULATE([Total Titles], KEEPFILTERS(netflix_titles[type]=_movie))` (`#,##0`, Catalog Metrics); `Total TV Shows` = `VAR _tv="TV Show" RETURN CALCULATE([Total Titles], KEEPFILTERS(netflix_titles[type]=_tv))` (`#,##0`, Catalog Metrics); `% Movies` = `DIVIDE([Total Movies],[Total Titles])` (`0.0%`, `Catalog Metrics \ Ratios`); `% TV Shows` = `DIVIDE([Total TV Shows],[Total Titles])` (`0.0%`, `Catalog Metrics \ Ratios`); `Genre Rank` = `RANKX(ALL(netflix_titles[listed_in]),[Total Titles],,DESC,DENSE)` (`#,##0`, `Helper \ Ranking`). (Depends on T007.)
- [ ] T010 Create `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/relationships.tmdl` with relationship 2: `DimDate[Date]` → `netflix_titles[Year]`, one-to-many, **single** cross-filter direction, active. (Depends on T007, T008.)

**Checkpoint**: The shared model (catalog + date dimension + measures + date relationship) exists. `[Total Titles]` resolves and the year axis is available.

## Phase 3: User Story 1 - Dynamic Row-Level Security by Country (Priority: P1) 🎯 MVP

**Goal**: Each viewer sees only the catalog titles for the country mapped to their sign-in identity, generalizing the source workbook's hardcoded `user2@maq.com` predicate to all users via the entitlement table.

**Independent Test**: Impersonate `shashank@maq.com` (India), `user2@maq.com` (United States), `user3@maq.com` (United Kingdom), and an unknown user; confirm each sees only their country's titles and the unknown user sees zero (SC-002/SC-003).

### Implementation for User Story 1

- [ ] T011 [US1] Create `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/tables/User_Access.tmdl` — entitlement table. M partition loads `Data/Netflix RLS/User_Access.csv` via `Csv.Document(File.Contents(...), [Delimiter=",", Encoding=65001, QuoteStyle=QuoteStyle.Csv])`, promote headers, cast `Username`/`Country` → `type text`, apply `Text.Trim` to `Country` (join key). **Do NOT deduplicate** (multiple usernames may map to one country — star-schema Limitations §4). Declare columns `Username`, `Country`.
- [ ] T012 [US1] Add relationship 1 to `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/relationships.tmdl`: `User_Access[Country]` → `netflix_titles[country]`, one-to-many, **single** cross-filter direction (User_Access → netflix_titles), active. Must **not** be bidirectional (preserves the RLS guarantee — plan Complexity Tracking). (Depends on T007, T010, T011.)
- [ ] T013 [US1] Create the dynamic RLS role `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/roles/Dynamic Country Access.tmdl` with `modelPermission: read` and `tablePermission User_Access = User_Access[Username] = USERPRINCIPALNAME()`. Filter MUST be a pure row-level boolean with **no measure reference** (constitution §4a). (Depends on T011.)
- [ ] T014 [US1] Register `User_Access` (`ref table`) and the `Dynamic Country Access` role in `Output/NetflixRLS/NetflixRLS.SemanticModel/definition/model.tmdl`. (Depends on T006, T011, T013.)
- [ ] T015 [US1] **Semantic model validation** — run `& "plugins\pbip\hooks\bin\tmdl-validate-windows-x64.exe" "Output\NetflixRLS\NetflixRLS.SemanticModel\definition"` then `python "plugins\pbip\skills\pbip\scripts\validate_pbip.py" "Output\NetflixRLS"`. Confirm: both relationships many-to-one/single-direction/active, RLS relationship NOT bidirectional, role parses with `modelPermission: read` and references only `User_Access[Username]`/`USERPRINCIPALNAME()`, `DimDate` marked as date table. Fix all exit-code-2 errors before proceeding. (Depends on T005–T014.)

**Checkpoint**: The complete secured semantic model validates clean; RLS propagation reasoning holds for all three sample users and the empty-set case.
